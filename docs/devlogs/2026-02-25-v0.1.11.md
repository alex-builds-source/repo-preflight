# repo-preflight v0.1.11

## What shipped
- Synced package metadata versioning by aligning `pyproject.toml` with runtime `__version__`.
- Added regression coverage to ensure project metadata and package version stay in lockstep.

## Why it matters
- Prevents release metadata drift that can break trust in `--version`, package builds, and downstream automation.
- Adds a guardrail so future releases fail fast if version files diverge.

## Validation
- `PYTHONPATH=src pytest -q` → `27 passed`
- `PYTHONPATH=src python -m repo_preflight.cli check --path . --strict --compact` → `pass=18 warn=0 fail=0`
- `gitleaks git --redact` → no leaks found
